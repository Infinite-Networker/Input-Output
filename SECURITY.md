## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability in Input-Output, please report it responsibly:

1. **GitHub Private Advisory** *(preferred)*:
   [https://github.com/Infinite-Networker/Input-Output/security/advisories/new](https://github.com/Infinite-Networker/Input-Output/security/advisories/new)

2. **Email**: Contact the repository maintainers through GitHub (@Infinite-Networker)

### What to Include

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any proof-of-concept code (if applicable)
- Suggested mitigations (if known)

## Response Timeline

| Action | Timeframe |
|:-------|:---------:|
| Initial acknowledgment | 48 hours |
| Severity assessment | 5 business days |
| Fix development | Depends on severity |
| Public disclosure | After patch is released |

---

## Scope

The following are **in scope** for security reports:

- Authentication or authorization flaws in the API
- Data exposure vulnerabilities (user data, API keys)
- SQL injection, XSS, CSRF vulnerabilities
- Dependency vulnerabilities with a CVSS score ≥ 7.0

The following are **out of scope**:

- Rate limiting on public endpoints
- Missing security headers on static demo pages
- Theoretical vulnerabilities without proof of concept
