## 🌟 Ways to Contribute

- 🐛 **Report bugs** — open an issue with the `bug` label
- 💡 **Suggest features** — open a discussion or issue with the `enhancement` label
- 📖 **Improve documentation** — fix typos, add examples, clarify explanations
- 🔌 **Add repository connectors** — help us index more open-access sources
- 🧪 **Write tests** — increase coverage across the codebase
- 🌍 **Translations** — help make Input-Output accessible in more languages

## 🚀 Getting Started

### 1. Fork & Clone

```bash
git clone https://github.com/YOUR_USERNAME/Input-Output.git
cd Input-Output
git remote add upstream https://github.com/Infinite-Networker/Input-Output.git
```

### 2. Create a Branch

```bash
git checkout -b feature/your-amazing-feature
# or
git checkout -b fix/the-bug-you-fixed
```

### 3. Make Your Changes

Follow the code style and add tests for new functionality.

### 4. Commit with Conventional Commits

```bash
git commit -m "feat: add semantic search filter by DOI"
git commit -m "fix: correct APA citation for multi-author papers"
git commit -m "docs: add setup guide for Windows"
git commit -m "test: add unit tests for license classifier"
```

**Commit types:** `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### 5. Push & Open a PR

```bash
git push origin feature/your-amazing-feature
```

Then open a Pull Request on GitHub with a clear description of what you changed and why.

## 📋 Code Standards

- **Python**: PEP 8, type hints encouraged, docstrings required for public APIs
- **JavaScript/TypeScript**: ESLint + Prettier, JSDoc for public functions
- **CSS**: BEM naming convention, custom properties for theming
- **Tests**: Pytest (Python) / Vitest (JS) — maintain ≥ 80% coverage

## 🏷️ Issue Labels

| Label | Description |
|:------|:-----------|
| `bug` | Something isn't working |
| `enhancement` | New feature or improvement |
| `documentation` | Documentation only changes |
| `good first issue` | Great for newcomers |
| `help wanted` | Extra attention needed |
| `wontfix` | This will not be worked on |
